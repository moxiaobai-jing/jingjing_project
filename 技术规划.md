# 类 ChatGPT 对话问答系统 — 技术规划

## 系统架构总览

```
┌─────────────────────────────────────────────┐
│                  前端 (Web)                  │
│         React + TypeScript + Tailwind        │
└──────────────────┬──────────────────────────┘
                   │ HTTPS / SSE
┌──────────────────▼──────────────────────────┐
│               后端 API 层                    │
│              Python FastAPI                  │
├──────────────────────────────────────────────┤
│  认证服务  │  对话服务  │  文件服务（阶段二）│
└──────┬─────┴─────┬──────┴─────────────────────┘
       │           │
┌──────▼───┐ ┌────▼────────┐ ┌─────────────────┐
│ 用户DB   │ │  OpenAI API │ │ 向量数据库(可选) │
│(Postgres)│ │  (GPT 系列) │ │  Qdrant/Pinecone │
└──────────┘ └─────────────┘ └─────────────────┘
```

---

## 前端功能规划

### 核心功能

| 功能 | 说明 |
|------|------|
| 对话界面 | 气泡式消息列表，用户/AI 消息区分 |
| 流式输出 | 基于 SSE 实时打字机效果 |
| Markdown 渲染 | 支持代码高亮、表格、数学公式（KaTeX）|
| 会话管理 | 左侧侧边栏，历史会话列表、新建/删除/重命名 |
| 用户认证 | 登录/注册页，JWT token 持久化 |
| 消息操作 | 复制、重新生成、点赞/踩 |
| 模型切换 | 下拉选择 GPT-4o / GPT-3.5 等 |
| 系统提示词 | 自定义 AI 角色（System Prompt）|

### 技术选型

| 层面 | 技术 |
|------|------|
| 框架 | React 18 + TypeScript |
| 样式 | Tailwind CSS + shadcn/ui |
| 状态管理 | Zustand |
| 路由 | React Router v6 |
| 流式请求 | fetch ReadableStream / EventSource |
| Markdown | react-markdown + rehype-highlight |
| 构建工具 | Vite |

## 后端功能规划

### 核心模块

#### 1. 认证服务
- 注册/登录（邮箱 + 密码）
- JWT access token + refresh token
- 密码加密（bcrypt）

#### 2. 对话服务（核心）
- 会话（Conversation）CRUD
- 消息发送，调用 OpenAI API，流式返回
- 消息历史存储与读取
- 上下文截断（token 限制管理）
- 系统提示词配置

#### 3. LLM 接入层
- 基于 OpenAI Python SDK（`openai >= 1.0`）
- 流式响应封装为 SSE 推送给前端
- 错误重试、超时处理
- 用量统计与限流

#### 4. 文件服务（阶段二）
- 文件上传存储（本地 / S3）
- PDF 解析、文本提取
- 向量化 + 存入向量数据库
- RAG 检索增强生成

### 技术选型

| 层面 | 技术 |
|------|------|
| 框架 | Python FastAPI |
| LLM SDK | openai >= 1.0 |
| 数据库 | PostgreSQL（对话历史）+ Redis（缓存）|
| ORM | SQLAlchemy + Alembic（数据库迁移）|
| 认证 | python-jose（JWT）+ passlib（bcrypt）|
| 文件存储 | 本地 / AWS S3（阶段二）|
| 向量数据库 | Qdrant（阶段二）|
| 容器化 | Docker + Docker Compose |

## 数据库核心表设计

```sql
-- 用户表
users (
  id, email, hashed_password, created_at
)

-- 会话表
conversations (
  id, user_id, title, system_prompt, model, created_at, updated_at
)

-- 消息表
messages (
  id, conversation_id, role,   -- role: user / assistant / system
  content, token_count, created_at
)

-- 文件附件表（阶段二）
attachments (
  id, conversation_id, filename, file_path, created_at
)
```

## 本地启动方式

```bash
# 一键启动所有服务（后端 + 前端 + PostgreSQL + Redis）
docker-compose up

# 前端开发模式
cd frontend && npm run dev       # 访问 http://localhost:5173

# 后端开发模式
cd backend && uvicorn app.main:app --reload   # 访问 http://localhost:8000/docs
```
